# 邮箱账号（Outlook）

OctoManger 内置 Outlook 邮箱账号管理功能，独立于 OctoModule 体系，有专属的管理页面。

---

## 功能概览

| 标签页 | 功能 |
| --- | --- |
| **账号列表** | 查看、搜索、编辑、验证、删除邮箱账号 |
| **手动添加** | 通过 OAuth 弹窗授权，手动添加单个 Outlook 账号 |
| **批量导入** | 从文件批量导入账号凭据（access_token / refresh_token） |
| **批量注册** | 通过 Python 脚本批量生成并注册邮箱账号 |
| **OAuth Config** | 配置 Azure OAuth 应用参数（Client ID、Tenant 等） |

---

## 第一步：配置 OAuth

在使用手动添加或批量功能之前，需要先在 **OAuth Config** 标签页填写 Azure 应用配置。

| 字段 | 说明 |
| --- | --- |
| `Client ID` | Azure 应用的 Application (client) ID |
| `Client Secret` | 仅手动 OAuth 流程需要，批量导入不使用 |
| `Tenant` | `consumers`（个人账号）、`common`（通用）或具体 Tenant ID |
| `Redirect URI` | 与 Azure 应用注册时填写的一致，默认 `http://localhost:8080/oauth/callback` |
| `OAuth Scope` | 权限范围，默认包含 `Mail.Read` |
| `Mailbox` | 默认监听的邮箱文件夹，通常为 `INBOX` |
| `Graph Base URL` | Microsoft Graph API 地址，默认 `https://graph.microsoft.com/v1.0` |

配置会存储在数据库中，所有功能共用同一套 OAuth 配置。

---

## 账号状态

| 状态值 | 说明 |
| --- | --- |
| `0` 待验证 | 账号已添加，尚未验证能否正常读取邮件 |
| `1` 已验证 | 账号可以正常使用 |

---

## 手动添加（OAuth 弹窗）

1. 切换到 **手动添加** 标签页
2. 点击授权按钮，系统打开 Microsoft 登录弹窗
3. 用户在弹窗中完成 OAuth 授权
4. 系统从 callback 拿到 token，自动写入数据库

适合逐个添加个人账号，不适合批量场景。

---

## 批量导入

适合已经有账号凭据（access_token / refresh_token）的场景，从文件批量写入。

导入格式（CSV 或 JSON，具体格式参考控制台页面提示）。

---

## 批量注册

通过系统内置的 Go 注册器批量生成邮箱账号（不需要真实 OAuth 流程，适合生成测试数据或特定域名账号）。

| 字段 | 说明 |
| --- | --- |
| `服务商` | `outlook`、`hotmail`、`gmail` 等 |
| `数量` | 生成账号数，最大 200 |
| `前缀` | 邮件地址前缀，如 `user` |
| `域名` | 邮件域名，如 `example.com` |
| `起始序号` | 从哪个编号开始，默认 1 |
| `状态` | 创建后的账号状态，`0`=待验证，`1`=已验证 |

生成规则：`{前缀}{起始序号+i}@{域名}`

例如：前缀 `user`，域名 `example.com`，起始 1，数量 3，生成：
- `user1@example.com`
- `user2@example.com`
- `user3@example.com`

批量注册是**异步**操作，提交后立即返回 Job ID，在 **Jobs** 页查看执行结果。

详细的批量注册 API 文档见 [dev/邮箱批量注册](dev/邮箱批量注册.md)。

## 账号列表操作

### 查看邮件

在账号列表中点击 **眼睛图标** 打开账号详情，可以：
- 查看邮箱当前文件夹（Mailbox）
- 列出收件箱邮件列表
- 点击邮件查看完整内容（HTML / 纯文本）

### 编辑账号

点击 **铅笔图标** 编辑：
- 服务商（provider）
- 账号状态（待验证 / 已验证）
- Graph Config（JSON 格式，修改 OAuth token 等底层配置）

### 验证账号

点击 **双勾图标**，将账号状态标记为"已验证"。

### 批量操作

在列表中勾选账号，可以：
- **批量验证**：批量标记为已验证
- **批量删除**：不可恢复，操作前会弹窗确认

## Graph Config 说明

每个邮箱账号内部存储一个 `graph_config` JSON，包含 Microsoft Graph API 的认证信息：

```json
{
  "auth_method":    "graph_oauth2",
  "username":       "user@example.com",
  "tenant":         "consumers",
  "access_token":   "...",
  "refresh_token":  "...",
  "token_url":      "https://login.microsoftonline.com/consumers/oauth2/v2.0/token",
  "graph_base_url": "https://graph.microsoft.com/v1.0",
  "mailbox":        "INBOX"
}
```

控制台的账号详情页只展示摘要（不含原始 token），完整的 `graph_config` 在编辑页面（JSON 编辑器）中可见和修改。
