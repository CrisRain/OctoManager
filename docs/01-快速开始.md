# 快速开始

目标：让你在 5 分钟内把 OctoManger 跑起来，完成初始化，并走通最基本的账号管理流程。

---

## 前置要求

| 工具 | 最低版本 | 说明 |
| --- | --- | --- |
| Docker + Docker Compose | Docker 20+ | 推荐方式，一键启动所有依赖 |
| Go | 1.22+ | 仅本地开发时需要 |
| Bun | 1.x | 仅本地开发前端时需要 |
| Python | 3.10+ | 仅本地开发时需要 |

---

## 第一步：启动服务

### 方式 A：Docker Compose（推荐）

```bash
docker compose up -d --build
```

启动后可用的地址：

| 服务 | 地址 |
| --- | --- |
| 控制台 + API | http://localhost:8080 |
| PostgreSQL | localhost:5432 |
| Redis | localhost:6379 |

### 方式 B：本地开发

先启动数据库和缓存：

```bash
docker compose up -d postgres redis
```

再分三个终端分别启动后端、Worker、前端（顺序不限）：

```bash
# 终端 1：API 服务
cd backend && go run ./cmd/api

# 终端 2：Worker
cd backend && go run ./cmd/worker

# 终端 3：前端
cd web && bun install && bun dev
```

> 前端默认代理 `/api` 到同源，如果 API 在不同端口，需要设置 `VITE_API_BASE`：
> ```bash
> VITE_API_BASE=http://localhost:8080 bun dev
> ```

详细配置说明见 [03 · 部署与配置](03-部署与配置.md)。

## 第二步：初始化

首次访问 `http://localhost:8080` 时，系统会自动跳转到初始化向导（`/setup`）。

**步骤 1 — 运行数据库迁移**

点击"运行迁移"，系统执行 GORM AutoMigrate 创建所有数据表并写入默认系统配置。对已有数据库是安全的，可以重复执行。

**步骤 2 — 创建第一个 Admin Key**

输入密钥名称（如 `主管理员`），点击"创建 Admin Key"。

**步骤 3 — 保存密钥**

页面会显示完整密钥（格式如 `octo_xxxxxxxx...`），**只显示这一次**，请立即复制保存。密钥已自动写入当前浏览器会话，关闭前无需重新输入。

> 创建第一个 Admin Key 之后，所有 `/api/v1/*` API 都需要携带 `X-Api-Key` 请求头。
> 详见 [04 · 鉴权与 API Key](04-鉴权与API-Key.md)。

## 第三步：走通基本流程

下面是一个从零开始的最小流程演示，全程在控制台操作。

### 3.1 创建账号类型

打开 **Account Types** 页，点击"新建"：

```
key:      demo_shop
名称:     示例商店
category: generic
```

`schema`（定义账号表单字段）：

```json
{
  "type": "object",
  "properties": {
    "username": { "type": "string", "title": "用户名" },
    "password": { "type": "string", "title": "密码" }
  },
  "required": ["username", "password"]
}
```

`capabilities`（定义支持的动作）：

```json
{
  "actions": [
    { "key": "REGISTER" },
    { "key": "VERIFY" }
  ]
}
```

保存后，系统自动在 `scripts/python/modules/demo_shop/` 生成脚手架脚本。

### 3.2 创建账号

打开 **Accounts**，类型选 `demo_shop`，填写：

```
identifier: demo-user-01
username:   alice
password:   pass-123
```

### 3.3 编写模块逻辑

打开 **Octo Modules → demo_shop**，在编辑器里填写业务代码，然后点 **Dry Run** 验证：

```
action:     REGISTER
identifier: demo-user-01
spec:       {"username": "alice", "password": "pass-123"}
params:     {"invite_code": "ABC123"}
```

Dry Run 不执行真实请求，专门用于调试。

### 3.4 创建任务

打开 **Jobs**，新建：

```
type_key:   demo_shop
action_key: REGISTER
params:     {"invite_code": "ABC123"}
```

勾选账号 `demo-user-01`，提交。Worker 会异步执行并把结果写回 Job 详情。

## 推荐上手顺序

1. Docker Compose 跑起来
2. 完成初始化向导
3. 创建 `generic` 类型 → 创建账号 → Dry Run → Job
4. 如果需要 Webhook 接入，再看 [08 · Trigger 触发器](08-Trigger触发器.md)
5. 如果需要 Outlook 邮箱，再看 [09 · 邮箱账号](09-邮箱账号.md)
6. 模块细节见 [06 · OctoModule 开发指南](06-OctoModule开发指南.md)
