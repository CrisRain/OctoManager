# 部署与配置

---

## Docker Compose 部署（推荐）

### 启动

```bash
docker compose up -d --build
```

首次构建会拉取镜像、编译 Go 二进制、打包前端，大约需要 2–5 分钟。

```bash
# 查看容器状态
docker compose ps

# 查看日志（实时跟随）
docker compose logs -f api
docker compose logs -f worker
```

### 持久化数据卷

| 卷名 | 挂载路径 | 内容 |
| --- | --- | --- |
| `pgdata` | PostgreSQL 内部 | 数据库文件 |
| `redisdata` | Redis 内部 | Redis 持久化 |
| `octo_modules` | `/app/scripts/python/modules` | OctoModule 脚本、requirements.txt、各模块 .venv |
| `pip_cache` | pip 缓存目录 | pip 下载缓存，加速依赖安装 |

> 模块脚本存在 `octo_modules` 卷里，容器重建后不会丢失。

### 停止与清理

```bash
# 停止，保留数据
docker compose down

# 停止并删除所有数据卷（会清空数据库和模块脚本，谨慎！）
docker compose down -v
```

## 本地开发

适合需要热更新、断点调试、直接编辑 Python 模块的场景。

### 第一步：启动数据库和缓存

```bash
docker compose up -d postgres redis
```

### 第二步：启动后端 API

**必须在 `backend/` 目录下运行**，配置文件路径解析以此为基准：

```bash
cd backend
go run ./cmd/api
```

API 默认监听 `0.0.0.0:8080`。

### 第三步：启动 Worker

```bash
cd backend
go run ./cmd/worker
```

如果不希望 Worker 每次启动都重跑数据库迁移：

```bash
# Bash / PowerShell
DATABASE_AUTO_MIGRATE=false go run ./cmd/worker
$env:DATABASE_AUTO_MIGRATE = "false"; go run ./cmd/worker
```

### 第四步：启动前端

```bash
cd web
bun install   # 首次或依赖变更后运行
bun dev
```

前端默认在 `http://localhost:5173`，开发服务器自动代理 `/api` 到同源（`localhost:5173`）。

如果 API 在不同端口，需要指定 `VITE_API_BASE`：

```bash
# Bash
VITE_API_BASE=http://localhost:8080 bun dev

# PowerShell
$env:VITE_API_BASE = "http://localhost:8080"; bun dev
```

### 构建与测试

```bash
# 后端测试
cd backend && go test ./...

# 前端生产构建
cd web && bun run build
```

## 环境变量 / 配置文件

配置优先级：**环境变量 > `configs/config.yaml` > 代码默认值**

环境变量使用 `_` 作为层级分隔符，例如 `DATABASE_AUTO_MIGRATE` 对应配置文件的 `database.auto_migrate`。

### 数据库

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `DATABASE_URL` | （必填）| PostgreSQL 连接串，如 `postgres://user:pass@localhost:5432/octomanger` |
| `DATABASE_AUTO_MIGRATE` | `true` | 启动时自动执行 GORM AutoMigrate 和系统配置初始化 |
| `DATABASE_RESET` | `false` | `true` 时删除遗留表和列，**生产环境禁止开启** |

### Redis / Asynq

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `REDIS_ADDR` | `localhost:6379` | Redis 地址 |

### Python 执行桥

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `PYTHON_BIN` | `python` | Python 可执行文件路径 |
| `PYTHON_SCRIPT` | `../scripts/python/account_manager.py` | 旧版邮箱批量注册脚本入口（相对 `backend/` 目录） |
| `PATHS_OCTO_MODULE_DIR` | `../scripts/python/modules` | OctoModule 脚本根目录 |

### HTTP 服务

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `SERVER_ADDR` | `:8080` | API 监听地址 |
| `SERVER_WEB_DIST_DIR` | （空）| 指定后，API 服务同时托管前端静态文件。Docker 中设为 `/app/web/dist` |

### 前端构建变量

| 环境变量 | 默认值 | 说明 |
| --- | --- | --- |
| `VITE_API_BASE` | （空，即同源）| 前端请求 API 的基础 URL。生产部署同源时留空即可 |

## Docker Compose 默认配置说明

Docker Compose 中 API 和 Worker 容器已预设以下环境变量：

```yaml
DATABASE_URL:             postgres://octomanger:octomanger@postgres:5432/octomanger
REDIS_ADDR:               redis:6379
PATHS_OCTO_MODULE_DIR:    /app/scripts/python/modules
PYTHON_SCRIPT:            /app/scripts/python/account_manager.py
SERVER_WEB_DIST_DIR:      /app/web/dist
DATABASE_AUTO_MIGRATE:    true
```

如需修改，在 `docker-compose.yml` 的 `environment` 块中覆盖，或创建 `.env` 文件。

## config.yaml 示例

```yaml
server:
  addr: :8080
  web_dist_dir: ""

database:
  url: "postgres://octomanger:octomanger@localhost:5432/octomanger"
  auto_migrate: true
  reset: false

redis:
  addr: "localhost:6379"

python:
  bin: python
  script: "../scripts/python/account_manager.py"

paths:
  octo_module_dir: "../scripts/python/modules"
```
